## 二、重要提醒（升级前必读）

### 2.1 🔴 安全更新（高优先级）

**Gateway/WebSocket 跨站劫持漏洞修复**（GHSA-5wcw-8jjv-m286）

- **问题**：在 `trusted-proxy` 模式下，未严格校验浏览器来源，可能导致恶意站点劫持 WebSocket 连接，获取 `operator.admin` 权限。
- **修复**：对所有浏览器发起的 WebSocket 连接**强制执行 Origin 校验**，不再依赖代理头。
- **建议**：使用 Gateway + 反向代理的部署方式，**强烈建议尽快升级**。

### 2.2 ⚠️ 破坏性变更

**Cron 通知机制收紧**（#40998）

- **变更**：Cron 任务不再支持通过「临时 Agent 发送」或「主会话摘要回退」进行通知。
- **影响**：依赖上述方式实现 Cron 通知的配置将失效。
- **迁移**：运行 `openclaw doctor --fix` 可迁移旧版 Cron 存储及 `notify` / `webhook` 相关元数据。
- **建议**：若使用 Cron 通知，升级后先执行 `openclaw doctor --fix`，并检查 cron 配置是否符合新的隔离 delivery 要求。

## 三、核心亮点

### 3.1 安全与权限（多项加固）

除 WebSocket 漏洞修复外，本版本还包含一系列安全收紧：

| 领域 | 要点 |
|------|------|
| **Gateway 认证** | 本地 `gateway.auth.*` SecretRef 不可用时**拒绝启动**，不再静默回退到 `gateway.remote.*` |
| **Commands/config** | 强化 `/config`、`/allowlist` 的 `configWrites` 检查，阻止跨账号写入 |
| **系统执行** | `system.run` 在需要 approval 时，若无法绑定唯一本地文件操作数，则**拒绝执行** |
| **会话重置** | 将会话 `/new`、`/reset` 从 admin 级 `sessions.reset` 分离，防止通过 agent 路径越权 |
| **Plugin 权限** | 未认证的 plugin HTTP 路由不再继承 admin scope，`sessions.delete` 等仍受保护 |
| **session_status** | 沙箱子代理无法查看父会话元数据或写入父模型覆盖 |
| **nodes 工具** | `nodes` agent 工具改为 owner-only，非 owner 无法通过共享工具集访问 |
| **外部内容标记** | 支持空格分隔的 `EXTERNAL UNTRUSTED CONTENT` 边界，避免 prompt 包装绕过 sanitization |
| **密钥文件** | 强化 Secret 文件读取（禁止 symlink、路径交换），TAR 安装防止符号链接逃逸 |
| **SecretRef** | 拒绝 exec SecretRef 的 traversal ID 滥用 |
| **子代理** | 子代理在 spawn 时固化 leaf/orchestrator 控制域，防止恢复后重新获得编排权限 |

### 3.2 模型与 Agent 体验

- **Ollama 首类支持**：支持 Local / Cloud+Local 模式、浏览器云登录、模型推荐，云端模型避免多余本地拉取。
- **OpenCode Go**：新增 OpenCode Go provider，Zen 与 Go 共享同一 OpenCode 设置，单一 key 管理。
- **OpenRouter**：临时加入 Hunter Alpha、Healer Alpha 模型，便于试用（约一周窗口）。
- **Memory 增强**：支持 `memorySearch.extraPaths` 的多模态（图片/音频）索引，使用 `gemini-embedding-2-preview`，支持可配置维度和自动重索引。
- **文本 sanitization**：剥离 GLM-5、DeepSeek 等模型的控制 token（`<|...|>` 及全角变体），避免泄露给最终用户。

### 3.3 渠道体验（Telegram、Discord、飞书、Mattermost、MS Teams）

- **Discord**：自动创建线程支持 `autoArchiveDuration`（1 小时 / 1 天 / 3 天 / 1 周）；回复 chunking 修复。
- **Telegram**：HTML 长消息分块、预览与最终回复逻辑修复、exec approvals 防误匹配。
- **飞书**：本地图片路径正确传递 `mediaLocalRoots`，恢复图片上传能力。
- **Mattermost**：Markdown 保持首行缩进，表格原生渲染；plugin 发送 `replyTo` 逻辑修复。
- **MS Teams**：团队/频道 allowlist 使用 General 频道 conversation ID 作为 team key，匹配更可靠。
- **Signal**：严格 schema 下支持 `accountUuid`，loop-protection 配置不再报错。

### 3.4 ACP / IDE 集成

- `sessions_spawn` 支持 `resumeSessionId`，可继续已有 ACPX/Codex 对话。
- 会话恢复：`loadSession` 时回放用户/助手文本，展示 Gateway 控制的元数据。
- 工具流：`tool_call` / `tool_call_update` 增加文本与文件位置信息。
- 图片附件：支持将 inbound 图片附件转进 ACP 运行时。
- 主会话 alias：`main` 别名规范化，重启后 ACP 主会话可正确 rehydrate。
-  bundled `acpx` 升至 0.1.16，与最新 CLI 一致。

### 3.5 macOS / iOS

- **macOS**：Chat UI 增加模型选择器，thinking 级别可持久化；远程 Gateway 需要共享 token 时的检测与提示优化。
- **iOS**：Home canvas 新增欢迎屏与实时 Agent 概览，浮动控件改为固定工具栏，适配小屏；前台恢复时立即重连 Gateway。
- **TestFlight**：支持 Fastlane 的本地 beta 发布流程，watch-app 归档修复。

## 四、变更分类明细

### 4.1 安全与权限

| 变更 | 说明 |
|------|------|
| WebSocket Origin 校验 | 对所有浏览器发起的连接强制校验，修复 trusted-proxy 下跨站劫持 |
| Gateway SecretRef | 本地 auth SecretRef 不可用时 fail-closed |
| configWrites | 校验发起账户与目标账户，防止跨账号 config / allowlist 写入 |
| system.run | 无法绑定唯一文件时 fail-closed |
| 会话 /new、/reset | 从 sessions.reset 分离，防止通过 agent 越权 |
| Plugin 未认证路由 | 不再继承 admin scope |
| session_status | 沙箱子代理无法访问父会话 |
| nodes 工具 | 仅 owner 可调用 |
| 外部内容标记 | 支持空格分隔 EXTERNAL UNTRUSTED CONTENT |
| 密钥文件 | 禁止 symlink、路径交换；TAR 安装防符号链接逃逸 |
| SecretRef exec | 拒绝 traversal id 滥用 |
| 子代理权限 | 固化 leaf/orchestrator 作用域 |
| Telegram /approve | 禁止对其他 bot 使用，避免 ID 前缀误匹配 |

### 4.2 Agent 与模型

| 变更 | 说明 |
|------|------|
| 控制 token 剥离 | 移除 GLM-5、DeepSeek 等模型控制 token |
| Azure OpenAI Responses | 将 azure-openai 纳入 store override，修复 cron/embedded 400 |
| 错误渲染 | 成功轮次忽略过期的 errorMessage |
| 计费恢复 | 单 provider cooldown 探测，充值后可恢复无需重启 |
| Fallback | HTTP 499、Venice 402、Poe 402、Gemini MALFORMED_RESPONSE 等纳入可重试/fallback |
| Cooldown | 无失败记录时使用 unknown 而非 rate_limit |
| Auth cooldown | 过期后重置错误计数，避免循环放大 |
| Memory flush | memoryFlushWritePath 正确传到 embedded agent |
| Context 修剪 | 软 trim 时修剪仅图片的 tool result，与 tool-result 契约对齐 |
| Kimi Coding | anthropic-messages 工具以原生 Anthropic 格式发送 |
| 阿里云 Model Studio | MODELSTUDIO_API_KEY 支持 env 与 shell 回退 |
| 模型 input 能力 | 可选 model.input 检查防护 (#42096) |

### 4.3 渠道

| 渠道 | 变更 |
|------|------|
| Discord | autoArchiveDuration 可配置；maxLinesPerMessage、chunkMode 修复 |
| Telegram | HTML 分块；预览/最终回复与清理逻辑修复；exec approvals 与网络 env-proxy 修复；直接发送桥接到 message:sent hooks；轮询关闭时定时器清理 |
| 飞书 | mediaLocalRoots 传至 sendText，恢复本地图片上传 |
| Mattermost | 首行缩进保持；表格原生渲染；replyTo 回退逻辑 |
| MS Teams | allowlist 使用 General 频道 conversation ID |
| Signal | accountUuid 被严格 schema 接受 |
| Telegram schema | editMessage、createForumTopic 被接受；groups vs groupAllowFrom 文档澄清 |
| Discord schema | autoThread 暴露到 guild-channel 配置 |

### 4.4 ACP

| 变更 | 说明 |
|------|------|
| resumeSessionId | 支持从已有 ACPX/Codex 会话继续 |
| sessions.patch | 允许 spawnedBy、spawnDepth |
| 停止原因 | state:error 映射为 end_turn 而非 refusal |
| setSessionMode | 失败时正确返回给客户端 |
| Bridge 模式 | 不支持 per-session MCP 时显式拒绝 |
| loadSession | 回放文本、展示 controls/metadata |
| tool_call 事件 | 增加文本与文件位置信息 |
| 图片附件 | 传入 ACP 运行时 |
| acpx | 升级到 0.1.16 |
| main 别名 | 规范化后正确 rehydrate |
| sessions_spawn 流式 | mode=run 时对 orchestrator 会话向父级流式 |

### 4.5 其他重要修复

- **Gateway Control UI**：dashboard auth token 改为 session 存储，刷新后保持认证。
- **macOS launchd**：重启时保持 LaunchAgent 注册，支持 config/hot reload 恢复。
- **LaunchAgent 安装**：目录与 plist 权限收紧。
- **Channels allowlists**：去除过时 matcher 缓存，配置变更即时生效。
- **Discord/Telegram**：SecretRef 在发送路径中正确解析。
- **Web search**：Brave llm-context 处理；OpenRouter Perplexity citation 从 annotations 恢复。
- **CLI**：skills JSON 剥离 ANSI/C1；Windows 表格 ASCII 边框；memory teardown 关闭 watcher。
- **Control UI Sessions**：窄屏下单列表格正确折叠。
- **Cron**：空/NO_REPLY 不误判为需重试；lastErrorReason 记录。
- **Sandbox 子代理**：workspace 正确传给子代理。
- **依赖**：刷新依赖，ACP session-config 写入加固。

## 五、升级建议

### 5.1 升级前

1. **备份**：备份 `~/.openclaw/` 配置与数据。
2. **阅读破坏性变更**：确认 Cron 通知方式是否需要迁移。
3. **执行迁移**：升级后运行 `openclaw doctor --fix` 处理 legacy cron 与 notify/webhook。

### 5.2 升级步骤

```bash
# 使用官方安装脚本（示例）
curl -fsSL https://openclaw.ai/install.sh | sh

# 或使用包管理器升级
# 具体命令以官方文档为准
```

### 5.3 升级后验证

- 检查 Gateway 与各渠道是否正常连接。
- 如有 Cron，验证 `openclaw doctor --fix` 执行结果，并确认通知是否按预期工作。
- 检查依赖 SecretRef、子代理、插件等的高级配置是否仍符合预期。
